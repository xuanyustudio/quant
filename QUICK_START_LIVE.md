# 🚀 快速启动实盘交易

## 📋 **使用已有配置直接启动实盘**

如果您已经完成了：
1. ✅ `npm run stat-arb:find-pairs` - 分析配对
2. ✅ `npm run stat-arb:backtest` - 回测策略
3. ✅ `npm run stat-arb:portfolio` - 生成投资组合

现在可以直接使用生成的配置文件启动实盘！

---

## 🔧 **第1步：配置API密钥**

在项目根目录创建 `.env` 文件：

```bash
# 币安API配置
BINANCE_API_KEY=你的API_Key
BINANCE_SECRET=你的Secret_Key

# 代理配置（国内用户必需）
HTTPS_PROXY=http://127.0.0.1:7897
```

⚠️ **安全提示：**
- 永远不要将 `.env` 文件上传到GitHub
- 只启用 **现货交易** 权限
- **不要启用提现权限**

---

## 🚀 **第2步：启动实盘交易**

### **方法1：使用 npm 命令（推荐）**

```bash
npm run stat-arb:live -- --config=./output/live_trading_config_1761619197473.json
```

### **方法2：直接使用 node**

```bash
node src/statistical-arbitrage/live-trading.js --config=./output/live_trading_config_1761619197473.json
```

## 📝 **监控要点**

### **每天必查（必须！）：**
- [ ] 账户余额是否正常
- [ ] 持仓情况
- [ ] 当日盈亏
- [ ] 是否触发止损

### **查看日志：**

```bash
# 查看交易日志
tail -f logs/trades.log

# 查看错误日志
tail -f logs/error.log

# 查看综合日志
tail -f logs/combined.log
```

## ⚠️ **重要提醒**

### **在启动前确认：**

1. **✅ 配置文件是最新的**
   ```bash
   # 查看配置文件时间戳
   ls -l output/live_trading_config_*.json
   ```

2. **✅ 资金分配合理**
   - 查看配置文件中的 `tradeAmount`
   - 确保不超过您的风险承受能力

3. **✅ API密钥权限正确**
   - 只启用现货交易
   - 不启用提现权限
   - 考虑设置IP白名单

4. **✅ 代理正常运行（国内用户）**
   ```bash
   # 测试代理
   curl -x http://127.0.0.1:7897 https://api.binance.com/api/v3/ping
   ```

5. **✅ 已完成小额测试**
   - 建议先用 $100-$200 测试 3-7 天
   - 确认执行效果符合预期

## 🚨 **紧急停止条件**

立即停止交易如果：

1. **连续亏损**
   - 单日亏损超过设定值
   - 连续3天亏损

2. **技术故障**
   - API频繁报错
   - 订单执行异常
   - 网络连接不稳定

3. **市场异常**
   - 流动性突然枯竭
   - 价格剧烈波动 (±30%)
   - 交易所系统故障

4. **风险暴露**
   - 总回撤超过15%
   - 持仓风险过大
   - 无法正常平仓

## 📞 **需要帮助？**

如果遇到问题：

1. **查看日志** → `logs/` 目录
2. **检查配置** → 配置文件是否正确
3. **测试连接** → API是否可以访问
4. **暂停交易** → 有疑问时立即停止

---

## 💡 **最佳实践**

1. **小额开始** → 不要一次性投入全部资金
2. **持续监控** → 至少每天检查2次
3. **记录交易** → 做好交易日志
4. **定期复盘** → 每周分析一次表现
5. **及时止损** → 严格执行风控规则
6. **保持冷静** → 不要情绪化交易
